Custom Triggers — Two Options

Where and how should business owners define custom triggers?

A

Free-text trigger name (simple)

When creating a sequence, the owner types any trigger name they want — "After consultation call", "Quote approved", "Project kicked off". These act like named manual triggers: the owner fires them from a contact's profile by clicking "Fire trigger → [name]". No webhook, no automation — just organized manual sends.

Trigger type field in sequence editor:
B

Webhook trigger (powerful)

Each sequence gets a unique webhook URL. External tools (CRMs, booking systems, Zapier) POST to it to auto-enroll a contact. The owner copies the URL from the sequence settings and wires it up to their other tools. Fully automated, but requires technical setup.

Webhook URL shown in sequence settings:
https://yourapp.com/api/webhooks/enroll/abc123xyz

Recommendation: Start with A (free-text custom trigger). It covers 90% of use cases for small businesses without any external setup. Webhook support (B) can be added in a later phase when users need deeper integrations.
